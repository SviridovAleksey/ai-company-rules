# AI-Assisted Development Standard

Корпоративный стандарт разработки с использованием ИИ-инструментов.

**Версия:** 0.9 (бета)

## О чём это

Стандарт описывает процесс разработки продуктов с использованием ИИ: от проработки требований до деплоя. Не привязан к конкретным инструментам и технологическому стеку.

Ключевая идея: ИИ выступает исполнителем, но не заменяет инженерную культуру, архитектурное мышление и ответственность специалиста.

## Основные концепции

**Три уровня проектов:**
- **L1 (Public)** - любые одобренные ИИ-инструменты
- **L2 (Confidential)** - одобренные облачные инструменты
- **L3 (Restricted)** - только self-hosted модели внутри периметра

**Семь фаз процесса:**
1. Проработка документации (BA → SA → Architect → QA → DevOps → Developer)
2. Построение Roadmap
3. Апрув-ревью документации
4. Разработка по TDD с AI code review
5. Саппорт и входящие обращения
6. Процесс исправления багов
7. Рефакторинг и технический долг

**Ключевые принципы:**
- ИИ управляется документацией - качество продукта зависит от качества документации
- Принцип единственного владельца - каждый MD-файл имеет одного ответственного специалиста (человека)
- Writer/Reviewer паттерн - ИИ-ревью в отдельной сессии от разработки
- TDD без компромиссов - сначала тест, потом реализация
- Контекст как ресурс - управление контекстным окном ИИ

## Быстрый старт

1. Прочитать [основной документ](AI_STANDARD.md)
2. Определить уровень проекта (L1/L2/L3)
3. Вставить [обязательные правила](appendix-a-mandatory-rules.md) в AI_RULES.md проекта
4. Подключить [MCP-серверы](appendix-d-mcp-configs.md) по уровню проекта
5. Импортировать [toolkit](appendix-f-toolkit-repository.md) как git submodule
